【PHP】对、错与空

Publish: April 28, 2015

　PHP是一种被广泛应用的开源通用脚本语言，尤其适用于 Web 开发。

　条件判断是计算机语言实现分支的关键,在使用PHP的过程中，常会需要对变量是否存在或者其真假进行判断，博主针对PHP中的empty(),isset()，boolean 进行了一些总结! 　

一、Boolean

PHP中最简单的类型。表达了真值，可以为 TRUE 或 FALSE. 当变量被转换为 Boolean 时，以下值会被认为是 FALSE：

  • 布尔值 FALSE 本身
  • 整型值 0（零）
  • 浮点型值 0.0（零）
  • 空字符串，以及字符串 “0”
  • 不包括任何元素的数组 []
  • 不包括任何成员变量的对象

    仅 PHP4.0 适用

  • 特殊类型 NULL（包括尚未赋值的变量）
  • 从空标记生成的 SimpleXML 对象(博主未深入了解)

对应的，所有其他值均会被认做TRUE

表格内容比较多，很乱，但想明白上表中各Expression取值的规律，关键在于了解每个函数的意义和价值：

  • bool empty(mixed $var):

如果 var 是非空或非零的值，则 empty() 返回 FALSE。换句话说:’‘、0、’0’、NULL、FALSE、[]、var $var; 以及没有任何属性的对象都将被认为是空的，如果 var 为空，则返回 TRUE。

如果还没忘记的话，可以发现这里的定义和Boolean的定义是完全相反的，即Boolean与empty()是完全是取反的一对~ (有点意思~)

  • bool is_null ( mixed $var ):

如果 var 是 null 则返回 TRUE，否则返回 FALSE。

  • bool isset ( mixed $var [, mixed $... ] ):

如果 var 存在并且值不是 NULL 则返回 TRUE，否则返回 FALSE。

即 is_null() 和 isset() 是一对取反的冤家~

is_null(FALSE)自然是false， isset(FALSE)自然是TRUE啦.这么理解FALSE不是空，所以是有值的.

总结

注意上表红色部分，是博主自己在用PHP时被绕晕的几个小坑

  • int 0,string ‘0’与string ‘‘在竟然布尔值都是FALSE。。。

  • empty(FALSE) = true, empty() 与Boolean取反~
